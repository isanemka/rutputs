# Conventional Commit Message Template
# Example: feat(form): add quote request flow

# Types:
# feat:      A new feature
# fix:       A bug fix
# docs:      Documentation only changes
# style:     Code style changes (formatting, missing semi-colons, etc)
# refactor:  Code changes that neither fix a bug nor add a feature
# test:      Adding or updating tests
# chore:     Changes to build process, tooling, dependencies, etc.

# If needed, write a more detailed description below.
# Use the imperative mood: "add", "fix", "update", not "added", "fixed", "updated".
